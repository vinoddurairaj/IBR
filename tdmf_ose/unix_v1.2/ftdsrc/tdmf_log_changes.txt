--------------- 1.2.8p5+ Repair of Solaris panic --- Aug. 28, 2002 -- S.Wahl
  FTD_GET_GROUP_STARTED ioctl would panic the system under Solaris.
  Cause was that 1000 element array was member of structure passed to
  ioctl.  ioctl has size restriction of 256 bytes for the structure
  passed to it.  Solution was to externalize the array and replace 
  it with a pointer in the structure.  Have ioctl do a ddi_copyout
  of it's array contents to this user-land external array.
  File affected:
     driver/ftd_hpux.c
     driver/ftd_sun.c
     driver/ftd_ioctl.c
     driver/ftdio.h
     daemons/daemons/throtd.c
